## 📌 Project Overview
This project presents a complete **end-to-end data analysis** of Blinkit (online grocery platform) data containing 8,500+ records.  

The objective is to analyze **sales performance, customer satisfaction, and inventory distribution** to generate actionable business insights.

---

## 🎯 Business Problem
Retail businesses like Blinkit generate large volumes of data but often struggle to extract meaningful insights.

This project focuses on:
- Identifying key factors driving **sales performance**
- Understanding **customer behavior and satisfaction**
- Analyzing **inventory distribution across outlets**

## ⚙️ Project Workflow

### 🔹 1. Requirement Gathering
- Defined business objectives and KPIs  
- Focused on sales, customer satisfaction, and inventory  

---

### 🔹 2. Data Processing (SQL)
- Imported dataset into SQL Server  
- Cleaned data (fixed inconsistencies like category labels)  
- Performed analysis using:
  - `SUM()`, `AVG()`, `COUNT()`
  - `GROUP BY`, `ORDER BY`

---

### 🔹 3. Exploratory Data Analysis (Python)
- Used Pandas for data manipulation  
- Handled missing values and duplicates  
- Created visualizations to identify trends and patterns  

---

### 🔹 4. Dashboard Development (Power BI & Excel)
- Built interactive dashboards with:
  - KPI Cards (Sales, Ratings, Orders)  
  - Category-wise analysis  
  - Outlet performance  
  - Time-based trends  
- Added slicers for dynamic filtering

## 💡 Key Insights

- Customer ratings highlighted areas for operational improvement
